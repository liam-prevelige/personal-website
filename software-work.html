PreSell

PreSell is an e-commerce Android application that I've been building entirely on my own; it's certainly my largest independent project to date. After noticing how expensive merchandise with custom designs are - despite designers earning small margins on each sale - I decided to make my own marketplace. The app enables sellers to upload their unique designs onto the marketplace in a few clicks, selecting which types of goods (i.e. mugs, coasters, stickers, etc.) they'd like to sell with their design. All users can then explore and purchase these items in the feed. The purchasing process is unique; a sliding bar indicates the price the buyer would like to pay, with lower prices taking longer to sell/ship and higher prices selling immediately. A sale is made at the lower price point when enough other users request the item at the same or lower price point, reducing the item's unit cost. Although I still have more development to do before the app is available to the market (and sales can be made), I have sourced manufacturers and built the vast majority of the application, including the payments system (see below).

This has been an extremely fun, educational, and long project - I've been working on it off and on for a year and half. Since this app is built for Android, most of the code is written in Java. In total, there are 11 activities, 6 adapters, 10 fragments, 6 models, 6 other utilities, 40 layouts, and 6 menus. I host the database on Google Firebase and use Google's authentication, realtime database, storage, machine learning, and cloud functions. The payments system is handled via Stripe Connect; a Stripe account is created for a user when they sign up and placing orders is handled by a remote server and cloud functions. To maintain security, all financial information and payments processing is conducted through Stripe. Images are cached and loaded with Glide.

This app is stored in a private repo, but if you'd like to take a look, please reach out!

Intelligent Autonomous
Multirobot Patrol System

As one of two undergraduates in an upper level robotics course, I had the opportunity to pursue a loosely structured, research-oriented multirobot project. My teammate and I developed an intelligent patrolling system to detect robots escaping from a confined area.

My teammate and I developed a learning algorithm that adjusted the speed of patrolling robots as they detected escapees. The patrolling region is subdivided into a grid, with each cell containing a desired patroller velocity. Confined robots choose a random time and target exit (weighted by distance). If a patroller catches a robot escaping, the simulation pauses and the escapee returns to its starting location, while corresponding regions of the grid are updated according to our algorithm.

I used Docker to execute the Stage simulations in a Linux environment. Separate nodes, written in Python, exist for the patrollers, confined robots, and a 'security' system, which handles simulation metadata. The algorithm we developed improved detection up to 400%. We also wrote a formal research paper describing related work, the algorithm, simulation environment, and results, which can be viewed here.

Nuggets Server

Taking the lead in a team of three, I helped develop the "Nuggets" game server; players explore a set of rooms and passageways in search of gold nuggets with lifelike visibility limitations. The rooms and passages are defined by a map loaded by the server at the start of the game and gold nuggets are randomly distributed in piles within the rooms. Up to 26 players, with one spectator, may play a given game. After being randomly dropped in a room at the start of the game, players move about, collecting nuggets when they move onto a pile. When all gold nuggets are collected, the game ends and a summary is printed.

The code for this game's server is written in C and we used the Scrum project management methodology to organize the development process. By the time of completion, we had over 30 sprints and more than 300 commits to our git repository. In addition to taking a leadership role with the Scrum project management, I developed the infrastructure for the game's display and storage of the map, calculating a player's visibility, managing movement, collecting gold, conducting extensive unit and system testing, and creating documentation for design, implementation, and testing. This is a large, well-documented project; please reach out if you would like to see the repository!

Cancer Screening Classifier

Using MATLAB, I built a classifier to pre-screen patients for risk of ovarian cancer. The program builds a model based off of a set of training data, which consists of labeled spectrometry results divided by patients known to have ovarian cancer and known to not have it. The program takes the mean of spectra information from the training data for each group, averaging them out and storing the between-class mean. A scatter matrix is then computed within the class and between classes, allowing for the computation of the maximum generalized eigenvectors and the creation of a classification model to be applied to the test dataset. Based on the dataset I was given, the classifier was about 93% accurate in both the training and testing data, suggesting the classifier and corresponding eigenvector can be applied pretty generally; the ROC curves in training and testing were similar, so there is clearly consistency in spectra results among patients with and without ovarian cancer.

This project served as my introduction to classification models and applied computer science. To the left I have posted the ROC curves for training and testing respectively, as well as histograms for testing and training with averaged spectra results for those classified with and without ovarian cancer. Note that, at some thresholds, the histograms overlap, indicating the classifier would fail in these cases.

Ballistic Motion Authenticator

Since seeing viral videos of 'trick shots' - some of which seem too good to be true - I've always been interested in how to validate or invalidate their authenticity. For this project, I used MATLAB to write a program that can analyze videos of, say, a ball's trajectory, and determine the expected ballistic motion.

A user selects the center of the traveling ball in selected video frames. Then, four types of least-squares estimations are run and the best fitting regression is selected. The predicted points are then overlayed on the frame. I've given examples of the output in the picture to the right, with the graph displaying the compiled estimations. The program also outputs the error for each regression.

Miscellaneous Cool Software Projects

Between classes and random ideas, I've written a lot of small programs that I find fun but probably aren't intensive enough to require their own section. Below I've listed a few highlights!

In MATLAB, I wrote a program that removes distortion in an image of a flat planar surface. In short, I estimate a homography that describes the projection of a planar surface from the 3D world to the 2D image camera sensor, then apply the inverse homography to an image. I love the visual of the license plate - the second image is the output of my program, removing distortion and allowing you to clearly make out the letters from an otherwise unreadable image.

In another project, I used Python to write the 'Six Degrees of Kevin Bacon' game. If you're not familiar with this game, you choose a random actor/actress and see how many degrees of separation they lie from Kevin Bacon; i.e. Actor A was in Movie X with Actor B, and Actor B was in Movie Y with Kevin Bacon, so Actor A is two degrees of separation from Kevin Bacon. With my program, you can also reset the universe around a different actor.

Although the list could go on, the final project I'll talk about was a challenge problem from my Algorithms class. Given a string s[1:n] and a dictionary D(), we needed to quickly determine all possible ways of reconstituting the string with anagrams and return the corresponding list. So, running the program on 'dynamic,' the output is 13 possible anagrams and the list is [’cyma din’, ’cyma nid’, ’cad miny’, ’cany dim’,’cany mid’, ’cyan dim’,’cyan mid’, ’many cid’,’myna cid’, ’cay mind’, ’damn icy’, ’mand icy’, ’dynamic’]. This generally takes a lot of time, but I came up with an optimal dynamic programming algorithm: (1) represent the dictionary as a map with key = character frequency : value = list of original words, (2) dynamically take permutations of the string, (3) compare permutations with dictionary keys, adding original words to list if they exist.
